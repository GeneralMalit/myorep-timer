# 06 Node Editor

Use `00-shared-brief.md` as the base direction.

Create the mobile node editing flow for workout and rest nodes inside the session builder.

## Goal
- Replace the current desktop-offset modal feeling with a mobile-first editing surface.
- Make update, link, and save actions clear without overwhelming the user.

## Must Show
- Editing state for a workout node.
- Name field and core editable values.
- Notes field.
- Clear actions for updating the node and saving or linking to the workout library.

## Intent
- This should feel like a bottom sheet or full-screen mobile editor, not a desktop dialog squeezed onto a phone.
- The screen should feel comfortable even when fields are numerous.

## Notes
- You do not need to show every possible field variation in one screen, but the structure should clearly support the real editing depth.
- Prioritize workout-node editing, since it is the denser and more important case.
