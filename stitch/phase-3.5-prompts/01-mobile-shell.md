# 01 Mobile Shell

Use `00-shared-brief.md` as the base direction.

Create the mobile shell for `MyoRep Timer`.

## Goal
- Define the top-level mobile navigation pattern.
- Make the app feel focused instead of drawer-heavy or desktop-like.
- Show how users access setup, session builder, library, settings, and account/sync without permanent sidebar chrome.

## Must Show
- Compact top header.
- A clear drawer or library entry point.
- A dominant primary task area.
- A lightweight preview of how library/navigation expands.

## Intent
- This screen should establish the mobile IA for the rest of the app.
- Do not use bottom tabs.
- Prefer one primary content area with contextual access to secondary surfaces.

## Notes
- Library, settings, and account/sync should feel nearby but not always open.
- The shell should feel calm enough to support both workout setup and timer use.
